# 智能物料管理系统 项目技术文档 v1.0

> 项目代号：VibeRack（暂定，沿用 Vibe 品牌线，可改）
> 配套文档：《智能底盘 BLE 接口规格 v0.1》（协议细节以该文档为准）
> 状态：架构定版，进入开发阶段

---

## 1. 项目定位

面向创客、电子工作室、小型研发团队的桌面级物料管理系统。核心价值链：**扫码入库 → NFC 碰一下 → 找料点灯 → BOM 拣料 → 库存台账**。

本质上是将工业 Pick-to-Light 拣货系统（商用单工位上万元）桌面化、消费化，单槽位硬件成本压至个位数人民币。找料点灯是获客入口，BOM 拣料与库存台账是留存核心，数字孪生视图是体验与传播锚点。

一句话定义：**手机里的货架和墙上的货架一起亮灯。**

## 2. 系统架构总览

```
┌─────────┐  BLE/NFC ┌──────────────────────────┐
│ 手机 APP │◄────────►│ 批次单元 ×N（每单元25槽）   │
│ (中继)   │          │ nRF52811+NT3H2111+锂电    │
└────┬────┘          │ 底盘灯条 25×WS2812B        │
     │ 同步           │ 料盒×25（纯结构件，磁吸）    │
┌────▼────┐          └──────────────────────────┘
│ 云端台账 │  BLE     ┌──────────────────────────┐
│ 多人协作 │◄────────►│ 共享取料坞（v1.5，USB供电）  │
└─────────┘          │ 光电计数 + 称重             │
                     └──────────────────────────┘
```

设计三原则：

1. **料盒是哑件**——零电子、零接触点，所有电子件固定焊死在底盘上，可靠性与成本同时最优。
2. **硬件是绑定关系的单一事实源**——槽位→料号→数量存于 MCU flash，手机与云端是缓存与富数据层；换机重装可"从硬件恢复"全仓数据。
3. **传感器收敛到共享设备**——逐盒加传感器破坏功耗与成本模型，自动计数能力由一台共享取料坞提供。

## 3. 硬件设计

### 3.1 批次管理端（主控板）

| 模块 | 选型 | 要点 |
|---|---|---|
| 主控 | nRF52811 | BLE 5，低占空广播平均 ~15µA；DATA 引脚必须选 PWM 可映射 GPIO |
| NFC | NT3H2111 | I2C 接 MCU；FD 引脚接 GPIOTE PORT 事件用于碰醒；NDEF 存设备 URI（MAC+批次ID） |
| 供电 | 1S 锂电 3.7V | MCU 经 LDO（XC6206-3.3 类）供电（nRF 上限 3.6V）；灯条经 P-MOS 门控由锂电直供 |
| 灯条门控 | P-MOS | WS2812B 单颗静态电流 ~0.6mA，25 颗常耗 15mA+ 会杀死电池，**必须整体断电**；DATA 线断电期间保持低电平防倒灌 |

关键省料结论：锂电直供灯条（3.7V）使 WS2812B 数据高电平阈值（0.7×VDD≈2.6V）低于 MCU 的 3.3V 输出，**无需电平转换，无需 5V 升压**。

### 3.2 底盘与料盒

- **定位凸台 + 磁吸分工**：凸台对底盘做绝对定位（定节距，防止盒间累积公差导致灯位错位——25 盒累积可达 3-5mm，必须规避）；磁铁负责吸附力与手感。凸台管"准"，磁铁管"爽"，缺一不可。
- 灯条为一条 PCB 集成于底盘前缘，每槽位一颗 WS2812B，对应透光窗/导光柱。
- 料盒纯结构件 + 2 颗磁铁；磁力中等即可（盒内可能存干簧管、霍尔元件等磁敏感物料）。
- **磁吸链块移动特性**：相邻盒吸合成块，可整块平移。排序插入 = 掰开链条 + 后段整块后移一格 + 新盒落位，恒定 3 步。**每排末尾保留 1–2 空位**作推移余量；排满触发"扩展新底盘"购买提示（增长飞轮）。

### 3.3 共享取料坞（v1.5）

- USB 供电，一坞服务全部批次。
- 卷料计数：对射式光电对管（ITR9606 类）×2 正交布置测方向；料带定位孔 4mm 标准节距；按封装节距换算元件数（0402/0603 一孔两颗，IC 按 8/12/16mm 节距）。
- 称重：HX711 + 称重传感器，按皮重与单颗重量反算数量，覆盖散料与裁切料段，兼做盘点校准。
- 工作流：取盒 → 料带穿导槽拉出 → 计数自动扣账（碰 NFC 或扫标签识别物料）→ 放回。

### 3.4 成本估算（FDM 打样阶段，单批次 25 槽）

| 项目 | 估算 |
|---|---|
| 主控板（MCU+NFC+LDO+MOS+辅料） | ~25 元 |
| 锂电（2000mAh 软包/18650） | ~8 元 |
| 灯条 PCB + 25×WS2812B | ~18 元 |
| 底盘结构件 | ~15 元 |
| 料盒 ×25（结构件 + 磁铁） | ~50 元 |
| **合计** | **~116 元，折合单盒 ~4.6 元** |

注塑量产后结构件成本预计下降 50% 以上。取料坞独立 SKU，BOM 约 30 元。

## 4. 固件设计

### 4.1 状态机

| 状态 | 行为 | 平均电流 |
|---|---|---|
| 低占空广播（常态） | 1.5s 间隔广播，灯条断电，FD 中断武装 | ~15µA |
| 快速广播 | NFC 碰醒触发，100ms 间隔，30s 窗口 | ~200µA（短时） |
| 已连接 | GATT 读写；交互期 interval 30–50ms，空闲协商 400ms+latency | ~50µA |
| 点灯中 | MOS 导通，执行灯效，定时熄灭 | 25–30mA（秒级） |

迁移：常态 --NFC碰醒--> 快速广播 --连接/超时--> 已连接/常态；常态 --远程找料--> 已连接；已连接 --点灯指令/超时熄灭--> 点灯中往返；断开回常态。

**熄灭定时器独立于 BLE 连接**——手机进口袋断连，灯按时熄而非立即熄。

电量结论：日常加权平均 ~20µA，2000mAh 电池寿命瓶颈为电芯自放电而非固件，宣传口径"一年以上"非常保守。

### 4.2 实现红线（打板前已确认）

1. WS2812B **禁止 GPIO bit-bang**，必须 PWM 外设 + EasyDMA 整帧输出（SoftDevice 射频中断不可屏蔽，bit-bang 必花屏）。
2. 绑定记录走 **FDS 写穿**，禁止裸调 NVMC；断电不丢账是信任底线。
3. FD 唤醒用 **GPIOTE PORT 事件**（通道事件会让高频时钟常开，多耗十几 µA）；FD 信号软件去抖 50ms。
4. **OTA DFU 双区升级 v1 必带**；看门狗必开，复位后首包 Notify 携带复位原因码。
5. 广播厂商字段携带：批次ID + 电量 + 状态位 + table_seq（绑定表版本号），APP 免连接渲染全场状态。

## 5. 通信协议（摘要）

详见《BLE 接口规格 v0.1》。要点：

- 必备自定义服务：绑定表服务（操作码协议）、灯控服务（17B 单帧指令）、Device Health 服务（电量/复位原因/健康标志）；标准 BAS/DIS/DFU 为可选补充能力。
- 槽位记录 16B 定长：slot + 料号(10B ASCII，C=立创/M=自定义) + qty + flags + CRC8。
- **结构化操作在 MCU 端执行**：INSERT_AT / REMOVE_AT / MOVE_BLOCK 由固件完成平移重编号，杜绝两端分叉。
- 灯控模式：FIND（两段式：整条闪 2s → 单点常亮）/ PICK（掩码多点拣料）/ SORT（红=取出、绿=落位双掩码）/ STOCK_IN（入库绿呼吸）/ FX（彩蛋）。
- 所有指令 ≤17B，默认 MTU 23 可达；NFC NDEF URI：`lcscerp://device?mac=…&batch=…&ver=1`，沿用 APP 现有 NFC 路由体系。

## 6. APP 软件设计

### 6.1 APP 现有基础

扫立创标签入库（解析 pc/qty）、搜索/手动入库、库位推荐、第二属性多优先级排序、BOM 导入匹配（绑定关系持久化）、NFC 标签读写路由、蓝牙打印、本地备份导入导出。

### 6.2 改造项（按优先级）

**P0 — 统一容器模型**：合并现有"库位"与"盒子/盒层"两套平行账。位置 = 容器 + 槽位；容器类型：普通库位（1 槽退化）/ 收纳盒（N 层）/ **智能底盘（25 槽 + MAC + 电量 + 固件版本 + table_seq）**。库存记录唯一，指向（容器ID, 槽位号）。同步修正"数量 0 显示未知"的语义，并为库存记录增加**安全库存阈值字段**（驱动红灯、热力图、补货清单）。

**P0 — 操作语义层**：在转移/删除之上新增"插入到第 N 位 / 移除并合拢 / 区块平移"原子操作，调用对应 BLE 操作码，槽位号由系统批量重算。"流水"升级为带时间戳的操作日志，为多设备增量同步铺路（现状全量覆盖导入仅适用单机）。

**P1 — 数字孪生视图（APP 主界面）**：与实物 1:1 的格子网格。浏览态叠加库存健康热力（红=低库存、橙=湿敏临期、灰=空位）；点格子→实物灯亮；碰 NFC→跳转对应孪生页并自动连接；整理/拣料方案在孪生上动画预览。**先上只读版**（网格 + 热力 + 点亮三件事），重交互后迭代。

**P1 — 从硬件恢复**：设置页新增入口；逐台碰 NFC → READ_ALL 拉取绑定表 → 凭料号联网重建物料详情。卸载/换机数据不丢。

**P2 — BOM 拣料**：BOM 匹配结果页增加"开始拣料"，按底盘分组生成槽位掩码下发 PICK；逐项勾选重发掩码。"分配到空层位"升级为"分配到空槽位 + STOCK_IN 绿灯引导"。

**P2 — 标签功能降权**：保留打印能力，资源向孪生视图倾斜；智能底盘槽位不再需要纸标签，物料标签与非智能容器标签继续维护。

## 7. 核心算法

### 7.1 整理模式（排序重排）

输入：当前槽位序列；目标序 = 现有第二属性多优先级比较器的排序结果。
求解：对 diff 后的乱序段生成**最少块操作方案**——利用磁吸块平移特性，每次插入恒定 3 个物理动作（掰开/整块后移/落位），优先规划"最长已有序子序列保持不动、其余逐个插入"（LIS 思路），物理动作数 = (n − LIS长度) × 3。
执行：逐步下发 SORT 指令（红=取出位，绿=落位），每步用户确认后推进；全部完成后用 MOVE_BLOCK/INSERT_AT 落账，校验 table_seq 与全表 CRC。

### 7.2 台账漂移容忍策略

无传感器前提下数量为台账值，必然漂移。策略：低值料（阻容）只做阈值预警不追求精确，阈值设保守；高值料走严格确认流程；APP 周期性提示对触发红灯的槽位做小批量盘点；v1.5 后由取料坞计数/称重自动校准。**孪生视图明示"台账状态"而非物理真值。**

### 7.3 既有算法沿用

库位推荐（历史库位 > 类目聚集 > 封装相近 > 类目前缀映射 > 默认回退）、BOM 匹配（手动绑定 > Supplier Part > Manufacturer Part > 被动件 Footprint/Value 规则匹配）保持现有实现，匹配目标从库位扩展到容器+槽位。

## 9. 风险与待验证清单

| 风险 | 等级 | 对策/验证 |
|---|---|---|
| 灯位与盒位错位（累积公差） | 高 | 凸台绝对定位方案；打样后实测 25 盒满排偏移量 |
| 台账漂移导致信任崩塌 | 高 | 7.2 容忍策略；红灯阈值保守；v1.5 取料坞校准 |
| 多手机操作冲突 | 中 | table_seq + 操作日志；v1 明示单机为主，v2 做合并 |
| 电池寿命不及预期 | 中 | 打样后实测各状态电流，校核 20µA 预算 |
| 磁力选型（手感 vs 磁敏感物料） | 低 | 打样 A/B 几档磁铁；说明书提示磁敏感元件 |
| WS2812B 在 3.7V 下亮度/一致性 | 低 | 灯条打样实测；不行则换 3.3V 兼容款（SK6812 变体） |

## 10. 开发并行分工建议

- **硬件**：主控板原理图（先锁 PWM 引脚映射与测试点）→ 灯条 PCB → 底盘/料盒结构（凸台节距与灯位对齐为第一验证项）。

- **APP**：容器模型重构（P0）与孪生只读版可并行；BLE 层按接口规格打桩，硬件未回板前用模拟设备联调。
